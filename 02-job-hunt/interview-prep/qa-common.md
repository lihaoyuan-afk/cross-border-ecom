## 一、跨境电商基础知识

### Q1：Amazon FBA 和 FBM 的区别是什么？各适合什么场景？

**FBA（亚马逊配送）：** 卖家将货物提前发至亚马逊仓库，由亚马逊完成仓储、拣货、发货和售后。
- 优势：自动获得 Prime 标签、物流快（1-2天）、省人力
- 成本：仓储费 + 配送费，长期压库有长期仓储费
- 适合：标品、高周转、追求转化率的商品

**FBM（卖家自发货）：** 卖家自营或使用第三方仓发货。
- 优势：灵活可控，成本可压缩，适合大件或低频商品
- 适合：大件商品、定制品、新品测试期

**答题要点：** 提到 Prime 标签对转化率的影响，以及仓储费与周转率的关系。

### Q2：BSR 是什么？在选品中怎么用？

BSR（Best Seller Rank，畅销排名）是 Amazon 各品类下基于近期销量的动态排名，数字越小越畅销。

**选品应用：**
- BSR < 1000：竞争激烈但市场验证充分，需看评价数（门槛）
- BSR 1000-5000：市场健康，评价数通常在数百，可切入
- BSR > 10000：市场较小或新品，需谨慎评估月销量

**通过 BSR 估算月销量：** 各工具（Jungle Scout/Helium 10）提供 BSR→月销量换算，例如无线耳机类目 BSR=500 约对应月销量 300-500 单（需工具核实具体数据）。

**结合自身项目：** 我在选品分析工具中用公式 `月销量 = 类目基准值 / BSR^0.55` 估算，并在竞品监控系统中实时追踪 BSR 波动。

### Q3：Listing 优化的核心要素有哪些？

| 要素 | 要点 |
|------|------|
| 标题 | 核心关键词前置，80-200 字符，包含品牌+核心词+规格 |
| Bullet Points | 5 条，每条突出一个核心卖点，含次要关键词 |
| 描述/A+ | 详细展示使用场景和差异化，图文结合 |
| 图片 | 主图白底高清，7-9 张，含场景图/尺寸图/对比图 |
| 关键词后台 | 填充 Search Terms，放无法自然植入的关键词 |

**评分维度（自研工具）：** 标题长度、关键词覆盖率、Bullet 数量、图片数量、价格竞争力。

### Q4：ACoS 和 ROAS 分别是什么？如何判断广告效果好坏？

- **ACoS（Advertising Cost of Sale）= 广告花费 / 广告销售额 × 100%**
  - 例：花费 $100，带来 $400 销售额 → ACoS = 25%
  - 行业参考：20%-35% 属于健康区间（因品类差异较大）
  - ACoS 越低说明广告效率越高

- **ROAS（Return on Ad Spend）= 广告销售额 / 广告花费**
  - 是 ACoS 的倒数表示：ACoS=25% ↔ ROAS=4
  - ROAS > 3 一般可接受

**判断标准：** 需结合利润率，若利润率 30%，则 Break-even ACoS = 30%，低于此值说明广告盈利。

### Q5：什么是库存周转率？库存不足或过多分别有什么影响？

**库存周转率 = 销售成本 / 平均库存价值**（或用天数衡量：库存天数 = 库存量 / 日均销量）

- **库存不足（断货）：**
  - 失去 Buy Box
  - BSR 排名下滑，恢复需要时间和成本
  - 错过大促节点销售机会

- **库存过多：**
  - FBA 长期仓储费（存超 365 天）大幅增加
  - 占用资金，降低资金周转效率
  - 清仓降价损失利润

**运营实践：** 通常设置安全库存 = 补货周期 × 日均销量 × 1.2（20% 缓冲）。

## 二、数据分析能力

### Q6：如果某个 ASIN 的销量突然下滑 30%，你会怎么分析原因？

**分析框架（从外到内）：**

1. **外部竞争：** 竞品是否降价、推新品或拿到 BadBoy（大促品）？→ 看竞品 BSR 变化
2. **Listing 被修改：** 标题/图片/关键词是否被人恶意修改？→ 检查 Listing 变更记录
3. **广告表现：** 广告预算耗尽/竞价上升/ACOS 恶化？→ 查广告报告
4. **评价变化：** 新增差评导致评分下降？→ 检查近期评价
5. **库存问题：** 是否出现过断货？→ 查库存记录
6. **平台政策：** 是否有类目限制或 Listing 违规警告？

**结合工具：** 用数据看板对比销售额/会话数/转化率三条曲线，判断是流量问题还是转化问题。

### Q7：如何用 Excel 分析广告数据？

常用操作：
- **数据透视表：** 按 ASIN 汇总广告花费、销售额、ACoS、点击数
- **VLOOKUP：** 将广告报告中的 ASIN 与 Listing 数据合并对比
- **条件格式：** ACoS > 35% 标红，< 15% 标绿，快速识别高消耗词
- **IF 函数：** `=IF(ACoS>利润率,"亏损","盈利")` 快速判断每个广告活动状态
- **图表：** 折线图展示 ACoS 趋势，辅助判断广告优化效果

## 三、行为面试题（STAR 法则）

### Q8：描述一个你通过数据分析解决问题的经历

**Situation（背景）：**
在开发竞品监控系统时，发现 Dashboard 首页接口响应较慢，用户体验差。

**Task（任务）：**
诊断慢查询原因并优化，目标是将响应时间降到 200ms 以内。

**Action（行动）：**
- 使用 MyBatis 日志发现接口发出了 4 次独立 SQL 查询（今日预警数/未读数/近期价格下跌数/平均变化幅度）
- 将 4 次查询合并为 1 条带条件聚合的 SQL（`CASE WHEN` + `COUNT(*) FILTER`）
- 同时为全局预警配置加了 `@Cacheable` 缓存，避免每次请求重复查询

**Result（结果）：**
接口从 4 次 DB 往返降为 2 次，响应时间降低约 60%，Dashboard 加载明显流畅。
